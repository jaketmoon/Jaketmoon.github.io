使用技术栈：Go, Gin, MongoDB, and WebSocket

来源：https://dev.to/gbubemi22/building-a-simple-chat-application-with-go-gin-mongodb-and-websocket-2joo

首先是程序的入口

main

调用了 server 包中的 NewServer() 函数，实例化了一个 server 对象，这里可以认为它是整个应用服务器的核心。这个函数可能用来初始化 HTTP 服务器、路由以及其他服务。

server.ListenAndServe()是调用http包中的方法开始监听指定端口并启动服务器的函数，负责处理传入的 HTTP/WebSocket 请求。

server

Server 结构体定义了服务器所需的组件：

​ port：服务器监听的端口。

​ db：连接的 MongoDB 数据库。

​ ws：WebSocket 服务器实例。

在NewServer函数中

  1. 获取端口：从环境变量 PORT 中读取端口号，如果转换失败则默认为 8080。

​ 2. 数据库连接：通过调用database包中的 database.New() 函数连接 MongoDB。

​ 3. 服务初始化：初始化服务，包括 ConversationService 和 MessageService。这些服务可能分别处理聊天会话和消息相关的业务逻辑。

​ 4. WebSocket 服务器：通过 websocket.NewWebsocketServer() 创建 WebSocket 服务器实例，并将服务注入其中。启动 WebSocket 服务器通过 go ws.Run()，使其在后台运行。

5.	**创建 HTTP 服务器**：通过 http.Server 创建 HTTP 服务器，设置了端口、处理器（通过 RegisterRouter() 注册的路由），以及请求的超时时间配置。

有趣的是，这部分代码之所以选择先创建一个 Server 结构体，然后再通过该结构体为 http.Server 传递参数，而不是直接给http.Server传参数，主要是为了实现模块化设计，便于扩展性和可维护性。将一些核心模块放入自己创建的server中有利于之后的开发，要增加的时候直接增加一个字段就行了，在之后的一些方法中，是不能直接对http.Server定义方法的func (s *http.Server) RegisterRouter() http.Handler，这个就会报错。

​ 1. Gin 初始化：使用 gin.Default() 初始化 Gin 实例，默认开启了日志和恢复中间件。

​ 2. 基础路由：

​ • / 路由：通过 s.HelloWorldHandler 处理基础的 GET 请求，返回一个 “Hello World” 消息。

​ 3. 错误处理中间件：

​ • 使用middleware包中的 middleware.ErrorHandlerMiddleware() 处理全局错误，并定义了默认路由 NoRoute 处理 404 错误。

​ 4. 用户路由：

​ • 注册了几个与用户身份验证相关的 API 路由，如用户创建、登录、验证邮件等操作。

​ • 使用了 VerifyToken() 中间件来保护部分路由，确保用户经过身份验证后才能访问这些 API。

​ 5. WebSocket 路由：

​ • r.GET(“/ws”, …) 用于处理 WebSocket 连接，当请求 /ws 时，WebSocket 服务器通过 s.ws.HandleConnections() 处理该请求，实现实时聊天功能。

​ 一个简单的 Hello World API。请求这个路由时，会返回一个包含 “message”: “Hello World” 的 JSON 响应，状态码为 200 OK。

model

定义了一下需要的model

database

首先创建一个globalClient的全局指针，指向mongo.Client，所以在程序的任何一个地方都可以用，init() 函数在程序启动时自动执行，调用 DBinstance() 连接 MongoDB 并保存全局客户端。

在DBinstance() 函数中，通过从环境变量 DB_DATABASE 读取 MongoDB 的 URI，使用 mongo.Connect() 方法创建客户端连接，以及Ping() 方法用来验证连接是否成功，确保 MongoDB 可访问。

New() 函数：返回连接到名为 “Gomongodb” 的数据库实例。”Gomongodb” 或其他数据库名称是你在连接到 MongoDB 集群（Cluster）时指定的数据库名称。你可以在 MongoDB Atlas 的集群中通过代码连接到任何你指定的数据库。如果这个数据库还不存在，MongoDB 会在你首次向该数据库插入数据时自动创建它。

Service 结构体：封装了 MongoDB 客户端，并通过 OpenCollection() 方法返回指定名称的集合对象，用于后续的数据库操作。

OpenCollection() 方法的作用是简化访问 MongoDB 集合的操作。它接受一个集合名称作为参数，并返回与该集合关联的 mongo.Collection 实例。这样，你可以使用此方法轻松访问数据库中的不同集合，而不需要每次手动写出完整的数据库和集合的路径。例如，如果你想操作名为 users 的集合，只需调用 OpenCollection(“users”)，然后可以对这个集合执行插入、查询等操作。

为什么还需要new函数，直接使用globalclient不行吗，因为init函数不是程序开始就执行了吗

1.代码可读性和模块化：通过 New() 函数，你可以为不同的场景提供数据库实例，并且使得代码更易于理解和维护。

2.错误处理：New() 函数为连接过程增加了错误处理的逻辑，确保每次使用数据库时都能正确处理错误。

3.灵活性：如果未来有多个数据库或者客户端需要连接，不同的 New() 函数可以帮助实现更灵活的设计。
